What this repo teaches

A fork is not a copy-paste. It is a conversation.

Fork boldly

You do not need direct write access to start contributing. You fork, work independently, and still stay connected to the original project.

Branch with purpose

Small branches make ideas easier to test, review, and explain. Every change gets its own little runway.

Commit the plot twists

Commits are the story beats. They show what changed, why it changed, and how your experiment evolved over time.

Pull request the finale

A pull request is where the distributed workflow becomes collaboration: review, feedback, iteration, and merge.

A tiny workflow in motion

The classroom adventure, rendered as a commit trail.

1. Spot the upstream

Find the original repository and decide what you want to change.

2. Fork and clone

Make your own remote copy, then bring it down to your machine.

3. Branch and build

Keep the change isolated so experiments stay clean and reversible.

4. Push your version

Your fork becomes the public stage for your latest idea.

5. Open the pull request

Invite the original repository to review, discuss, and merge.

Pocket spellbook

Small commands, dramatic consequences.

git clone <repo-url>

Bring a remote repository into your local world.

git checkout -b <branch-name>

Create a fresh branch and step into it immediately.

git status

Ask Git what changed and what still needs attention.

git add -A

Stage the current set of changes for the next commit.

git commit -m "message"

Capture one meaningful step in your project story.

git push origin <branch-name>

Publish your branch to your fork on GitHub.
